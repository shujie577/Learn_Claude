# SourceTree 联动 GitHub 仓库 完整步骤总结

我帮你把从 0 到 1 的完整流程整理成了**傻瓜式分步操作**，你跟着一步步点就能搞定，再也不会手忙脚乱啦！

------

## 一、准备工作：本地文件夹变成 Git 仓库

### 1. 初始化本地仓库（让普通文件夹被 Git 识别）

1. 找到你的本地文件夹（比如 `C:\Users\Luminescent\Desktop\study`），先随便新建一个文件（比如 `test.txt`），写点内容。

2. 打开「命令提示符」，执行以下命令：

   ```bash
   # 进入目标文件夹
   cd C:\Users\Luminescent\Desktop\study
   # 初始化 Git 仓库（生成 .git 隐藏文件夹）
   git init
   ```

   看到 

   ```
   Initialized empty Git repository
   ```

    就说明成功了。

### 2. 用 SourceTree 添加本地仓库

1. 打开 SourceTree，点击顶部的 `Add` 按钮。
2. 浏览选择你刚才初始化的 `study` 文件夹，此时已经没有 “不是有效路径” 的错误了。
3. 仓库名称填 `study`，点击「添加」，本地仓库就被 SourceTree 识别了。

------

## 二、GitHub 端：创建远程仓库

1. 打开 GitHub 官网，登录你的账号。

2. 点击右上角的 

   ```
   +
   ```

    → 

   ```
   New repository
   ```

   ，新建仓库：

   - 仓库名：填和你本地一致的名字（比如 `Learn_Claude`）
   - 公开 / 私有：选 `Public` 即可
   - **注意：不要勾选「Add a README file」**（避免和本地仓库冲突）

   

3. 点击「Create repository」，创建完成后，复制仓库的 HTTPS 地址（格式是 `https://github.com/你的账号/仓库名.git`）。

------

## 三、本地提交：把文件交给 Git 管理

1. 回到 SourceTree，你会在「文件状态」里看到刚才新建的 `test.txt`。
2. 勾选这个文件，在右下角的输入框写提交信息（比如 `Initial commit: 第一次提交代码`）。
3. 点击「提交」按钮，完成本地提交。此时你的本地仓库就有了第一个提交记录，分支也真正创建出来了。

------

## 四、SourceTree 绑定远程仓库（关键步骤）

1. 在 SourceTree 顶部菜单栏，点击「远端」→「添加 / 移除远端」。
2. 点击「添加」：
   - 远端名称：填 `origin`（Git 默认的远程仓库名，别改）
   - URL / 路径：粘贴你刚才复制的 GitHub 仓库 HTTPS 地址
3. 点击「确定」，保存设置。

------

## 五、推送代码：把本地文件传到 GitHub

1. 回到 SourceTree 主界面，点击顶部的「推送」按钮。
2. 此时「要推送的分支」列表里，已经出现了你的本地分支（比如 `master`），勾选它。
3. 点击「推送」，第一次推送会弹出 GitHub 授权窗口，按提示登录你的 GitHub 账号，完成授权。
4. 推送完成后，刷新你的 GitHub 仓库页面，就能看到你提交的文件和记录了！

------

## 六、后续日常使用流程（提交→推送）

1. 写完代码 / 修改文件后，打开 SourceTree，在「文件状态」里勾选修改过的文件。
2. 写提交信息，点击「提交」，把修改保存到本地仓库。
3. 点击「推送」，把本地提交同步到 GitHub 远程仓库。
4. （如果团队协作）需要拉取别人的代码时，点击「拉取」按钮，同步线上仓库的最新内容。

------

## 避坑小贴士

1. 分支问题：如果出现 `main` 和 `master` 两个分支，推荐统一用 `main`，可以在 GitHub 仓库设置里把默认分支改成 `master`，或者在本地合并分支后删除多余分支。
2. 权限问题：如果推送报错，检查 SourceTree 里配置的 Git 邮箱和 GitHub 账号邮箱是否一致，或者重新授权 GitHub。
3. 强制推送：新手千万不要勾选「强制推送」，容易覆盖线上别人的代码，造成数据丢失。
