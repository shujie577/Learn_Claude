## 二、`main` 和 `master` 到底是什么？

它们都是 Git 的**默认主分支名称**，本质上是同一个东西，只是叫法不同，就像 “老大” 和 “主心骨” 的区别。

表格







| 名称         | 背景说明                                              | 常见场景                               |
| ------------ | ----------------------------------------------------- | -------------------------------------- |
| **`master`** | Git 诞生以来一直用的默认分支名，沿用了很多年          | 老项目、旧版本 Git、一些教程里常见     |
| **`main`**   | GitHub 在 2020 年开始推广的新默认分支名，目的是更中立 | 新项目、现在的 GitHub 仓库、新版本 Git |

你现在的仓库里同时出现了 `main` 和 `master` 两个分支，是因为：

- GitHub 新建仓库时，默认创建的是 `main` 分支（你在线上初始化了 README，所以生成了 `main`）。
- 而你本地初始化仓库时，Git 默认创建的是 `master` 分支，推送上去后，线上就多了一个 `master` 分支。

------

## 三、怎么处理现在的两个分支？

你现在的仓库里有两个主分支，虽然不影响使用，但容易搞混，推荐你这样处理：

### 方案 1：统一用 `main`（和 GitHub 默认保持一致，推荐）

1. 在 SourceTree 里把本地分支切换到 `main`，或者把 `master` 分支合并到 `main`。
2. 删掉本地和线上的 `master` 分支，以后都用 `main` 作为主分支。

### 方案 2：统一用 `master`（保持你本地习惯）

1. 在 GitHub 仓库的 `Settings → Branches` 里，把默认分支改成 `master`。
2. 删掉线上的 `main` 分支，以后都用 `master` 推送代码。

## 四、补充小知识

- 分支名本身没有任何特殊功能，你甚至可以叫它 `xiaoxuexi`，只是约定俗成用 `main` 或 `master` 代表 “主分支”。
- 你界面上的 `2 Branches` 就是指 `main` 和 `master` 两个分支，不用慌，只是个小细节，不影响你正常用仓库。
